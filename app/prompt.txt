# 🧪 Lager App – Prüf- & Fix-Plan

Du bist ein erfahrener Flutter-Entwickler. Ich habe eine fertig implementierte
Flutter Lager-App, die noch **nie manuell getestet** wurde. Hilf mir dabei,

## 🏗️ Architektur-Überblick

- **Plattformen:** Android (Prio 1), Web (Prio 2), Desktop/Linux (Prio 3)
- **Sync-Backend:** PocketBase (primär) – SQLite lokal auf Mobile/Desktop
- **Web:** Direktzugriff auf PocketBase (kein SQLite)
- **Nextcloud:** Optional, keine Prio

## ✅ Prüfreihenfolge (nach Priorität)

### 🥇 Phase 1 – Android (Kern-Features)

Teste und fixe in dieser Reihenfolge:

1. **App startet ohne Crash**
   - `flutter run -d android`
   - PocketBase Service initialisiert sich korrekt?

2. **PocketBase Verbindung**
   - Einstellungen → PocketBase URL setzen
   - Verbindungstest → grünes Häkchen?

3. **Artikel anlegen (Mobile)**
   - Neuer Artikel → alle Felder ausfüllen
   - Bild aufnehmen (Kamera)
   - Speichern → in SQLite lokal?
   - Sync → in PocketBase angekommen?

4. **Artikel bearbeiten**
   - Artikel öffnen → Felder ändern → Speichern
   - Sync korrekt?

5. **Artikel löschen**
   - Löschen → aus SQLite entfernt?
   - Sync → aus PocketBase entfernt?

6. **QR/Barcode Scanner**
   - Scanner öffnen → Kamera startet?
   - QR-Code scannen → Artikel gefunden?

7. **PocketBase Sync (Push/Pull)**
   - Artikel auf Gerät A anlegen
   - Sync auf Gerät B → Artikel erscheint?
   - Konflikt erzeugen → Conflict Resolution Screen?

8. **Import (CSV/JSON/ZIP)**
   - Testdatei importieren
   - Artikel korrekt in SQLite?
   - Sync zu PocketBase?

9. **Export (CSV/JSON/ZIP)**
   - Exportieren → Datei korrekt?
   - ZIP enthält Bilder?

10. **PDF Export**
    - Artikel-Liste als PDF exportieren
    - PDF korrekt gerendert?
