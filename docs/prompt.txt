## 2) App-Einstieg & Navigation (`main.dart`)

`main.dart` (~17 KB) ist der zentrale Orchestrator.

### Initialisierungsreihenfolge (`main()`)
1. `AppConfig.init()` вЂ” Runtime-Konfiguration laden (Web: window.ENV_CONFIG)
2. `AppConfig.validateForRelease() + validateConfig()`
3. `FlutterError.onError` + `PlatformDispatcher.instance.onError` вЂ” globale Fehler
4. `platform.initDesktopDatabase()` вЂ” SQLite-FFI-Init (nur Native)
5. `PocketBaseService().initialize()` вЂ” PB-Client aufbauen
6. `AppLockService().init()` вЂ” Biometrie-Service (nur Native)
7. `runApp(MyApp())`

### Screen-PrioritГ¤tskette (`_buildHome()`)
1. `ServerSetupScreen` в†’ keine PB-URL konfiguriert
2. Lade-Spinner в†’ Auth-Token wird geprГјft
3. `LoginScreen` в†’ nicht eingeloggt (auГџer `PB_DEV_MODE=1`)
4. `AppLockScreen` в†’ App gesperrt
5. `ArtikelListScreen` в†’ Normalzustand

### Sync-Lifecycle
- Initialer Sync nach Login/Setup via `SyncOrchestrator.runOnce()`
- Periodischer Sync alle 15 Minuten via `Timer.periodic`
- WLAN-Only-Sync konfigurierbar (`wifi_only_sync` in SharedPreferences)
- `AppLockService.onAppPaused()` / `onAppResumed()` via `WidgetsBindingObserver`

### Wichtige Architektur-Details
- **GlobalKey:** `GlobalKey<NavigatorState> navigatorKey` fГјr `ConflictCallback` im `SyncOrchestrator`
- **Sync-Init:** Startet erst nach erfolgreichem Login. Bei Resume wird DB explizit neu geГ¶ffnet (B-004)

### Sync-Invarianten (niemals brechen)
- **`uuid`** в†’ stabil, gerГ¤teГјbergreifend eindeutig, nie Г¤ndern
- **`remote_path`** в†’ Bindung an PocketBase Record-ID
- **`etag`** в†’ entspricht `updated`-Zeitstempel (ISO 8601) von PocketBase. `NULL` = Pending
- **Konflikt-Erkennung:** Vor jedem `PATCH` Remote-Status prГјfen. Abweichung в†’ `onConflictDetected`-Callback (B-005)
- `deleted = 1` в†’ Soft-Delete lokal в†’ Hard-Delete beim nГ¤chsten Push

### Sync-Ablauf
```
Push (LOCAL в†’ REMOTE):
  etag IS NULL AND deleted = 0:
    remote_path leer    в†’ POST (create)
    remote_path gesetzt в†’ PATCH (update)
  etag IS NULL AND deleted = 1:
    в†’ DELETE auf PocketBase в†’ lokal entfernen

Pull (REMOTE в†’ LOCAL):
  updated_at > last_sync в†’ upsert lokal
  lokal vorhanden, remote nicht в†’ deleteArtikel()

Nach Record-Sync:
  downloadMissingImages()      в†’ nur wenn lokal fehlt ODER 0 Byte (B-003)
  setBildPfadByUuidSilent()    в†’ setzt Pfad ohne updated_at zu Г¤ndern
```
