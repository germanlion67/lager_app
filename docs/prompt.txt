## 2) App-Einstieg & Navigation (`main.dart`)

`main.dart` (~17 KB) ist der zentrale Orchestrator.

### Initialisierungsreihenfolge (`main()`)
1. `AppConfig.init()` — Runtime-Konfiguration laden (Web: window.ENV_CONFIG)
2. `AppConfig.validateForRelease() + validateConfig()`
3. `FlutterError.onError` + `PlatformDispatcher.instance.onError` — globale Fehler
4. `platform.initDesktopDatabase()` — SQLite-FFI-Init (nur Native)
5. `PocketBaseService().initialize()` — PB-Client aufbauen
6. `AppLockService().init()` — Biometrie-Service (nur Native)
7. `runApp(MyApp())`

### Screen-Prioritätskette (`_buildHome()`)
1. `ServerSetupScreen` → keine PB-URL konfiguriert
2. Lade-Spinner → Auth-Token wird geprüft
3. `LoginScreen` → nicht eingeloggt (außer `PB_DEV_MODE=1`)
4. `AppLockScreen` → App gesperrt
5. `ArtikelListScreen` → Normalzustand

### Sync-Lifecycle
- Initialer Sync nach Login/Setup via `SyncOrchestrator.runOnce()`
- Periodischer Sync alle 15 Minuten via `Timer.periodic`
- WLAN-Only-Sync konfigurierbar (`wifi_only_sync` in SharedPreferences)
- `AppLockService.onAppPaused()` / `onAppResumed()` via `WidgetsBindingObserver`

### Wichtige Architektur-Details
- **GlobalKey:** `GlobalKey<NavigatorState> navigatorKey` für `ConflictCallback` im `SyncOrchestrator`
- **Sync-Init:** Startet erst nach erfolgreichem Login. Bei Resume wird DB explizit neu geöffnet (B-004)

### Dev-Mode
```bash
flutter run --dart-define=PB_DEV_MODE=1  # überspringt Login-Screen
```

#### Tabelle `sync_meta`
- `last_sync`: Unix-Timestamp ms des letzten erfolgreichen Syncs

### Sync-Invarianten (niemals brechen)
- **`uuid`** → stabil, geräteübergreifend eindeutig, nie ändern
- **`remote_path`** → Bindung an PocketBase Record-ID
- **`etag`** → entspricht `updated`-Zeitstempel (ISO 8601) von PocketBase. `NULL` = Pending
- **Konflikt-Erkennung:** Vor jedem `PATCH` Remote-Status prüfen. Abweichung → `onConflictDetected`-Callback (B-005)
- `deleted = 1` → Soft-Delete lokal → Hard-Delete beim nächsten Push

### Sync-Ablauf
```
Push (LOCAL → REMOTE):
  etag IS NULL AND deleted = 0:
    remote_path leer    → POST (create)
    remote_path gesetzt → PATCH (update)
  etag IS NULL AND deleted = 1:
    → DELETE auf PocketBase → lokal entfernen

Pull (REMOTE → LOCAL):
  updated_at > last_sync → upsert lokal
  lokal vorhanden, remote nicht → deleteArtikel()

Nach Record-Sync:
  downloadMissingImages()      → nur wenn lokal fehlt ODER 0 Byte (B-003)
  setBildPfadByUuidSilent()    → setzt Pfad ohne updated_at zu ändern
```

### PocketBase Collections
| Collection          | Beschreibung              | Lokales Pendant           |
|:--------------------|:--------------------------|:--------------------------|
| `artikel`           | Haupt-Artikeldaten        | ✅ SQLite `artikel`       |

## 6) Authentifizierung & App-Lock

### Login-Flow (M-009)
- `LoginScreen`: E-Mail/Passwort, Validierung, Loading-State
- Auto-Login: Token-Refresh via `PocketBaseService.refreshAuthToken()`
- Logout: `PocketBaseService.logout()` + Sync-Timer stoppen
- Dev-Mode: `--dart-define=PB_DEV_MODE=1` überspringt Login

### App-Lock (F-001/F-002, ab v0.8.2)
- Paket: `local_auth: ^3.0.1`
- Service: `AppLockService` (Singleton, SharedPreferences-Persistenz)
- Screen: `AppLockScreen` — automatischer Biometrie-Start via `addPostFrameCallback`
- Fallback: Geräte-PIN/Pattern wenn Biometrie nicht verfügbar
- Timeout: Konfigurierbar in Minuten (Slider im Settings-Screen, Standard: 5 Min)
- Lifecycle: `WidgetsBindingObserver` → `onAppPaused()` / `onAppResumed()`
- Probe-Auth: Bei Toggle-Aktivierung wird einmalig `authenticate()` aufgerufen
- Nur Native: `kIsWeb`-Guard in `main.dart`

## 7) Theming / UI-Regeln (Material 3)

### Dreistufige Konfiguration in `app/lib/config/`
- **`AppConfig`** — Technische Tokens (alle Werte zentral, keine Magic Numbers):
  - Spacing: `spacingXSmall(4)` … `spacingXXLarge(32)`, `spacingSectionGap(20)`
  - Border-Radien: `borderRadiusXXSmall(2)` … `borderRadiusXLarge(16)`
  - Icon-Sizes: `iconSizeXSmall(14)` … `iconSizeXXLarge(64)`
  - Stroke-Breiten: `strokeWidthThin(1)` … `strokeWidthThick(3)`
  - Opacity: `opacitySubtle(0.1)`, `opacityLight(0.2)`, `opacityMedium(0.3)`
  - Layout: `loginFormMaxWidth(400)`, `setupFormMaxWidth(480)`, `buttonHeight(48)`
  - Pagination: `paginationPageSize`, `paginationScrollThreshold`
  - Suche: `searchDebounceDuration(300ms)`, `searchResultLimit`
  - Bilder: `pbThumbGroesse`, `maxWidth(800px)`, `imageQuality(85)`
- **`AppTheme`** — Material 3, `ThemeMode.system`, Google Fonts Roboto, `ColorScheme.fromSeed`
- **`AppImages`** — Asset-Pfade, `hintergrundAktiv`-Flag

### Pflichtregeln
```dart
// ✅ Richtig
color: Theme.of(context).colorScheme.surface
padding: EdgeInsets.all(AppConfig.spacingLarge)
borderRadius: BorderRadius.circular(AppConfig.cardBorderRadiusLarge)
color.withValues(alpha: AppConfig.opacitySubtle)  // withOpacity() ist deprecated

// ❌ Falsch
color: Colors.white          // bricht Dark Mode
padding: EdgeInsets.all(16)  // Magic Number
color.withOpacity(0.1)       // deprecated API
```

### Farb-Semantik
| Bedeutung   | colorScheme-Token                                        |
|:------------|:---------------------------------------------------------|
| Oberfläche  | `surface` / `onSurface`                                 |
| Sekundärtext| `onSurfaceVariant`                                      |
| Fehler      | `error` / `errorContainer` / `onErrorContainer`         |
| Erfolg      | `tertiary` / `tertiaryContainer` / `onTertiaryContainer`|
| Warnung     | `secondary` / `secondaryContainer` / `onSecondaryContainer` |
| Info/Primär | `primary` / `primaryContainer` / `onPrimaryContainer`   |

## 9) Logging (AppLogService) — Pflicht

```dart
final _log = AppLogService.logger;

_log.t('trace')
_log.d('debug')
_log.i('info')
_log.w('warning')
_log.e('error', error: e, stackTrace: st)
_log.f('fatal', error: e, stackTrace: st)
```

**Regeln:**
- Keine neuen `print()` / `debugPrint()` einführen
- Fehler immer mit `error:` + optional `stackTrace:`
- `AppErrorHandler.classify()` für automatische Exception-Klassifizierung
