## 🔑 API-Authentifizierung (PocketBase 0.36.x)

In PocketBase 0.36.x sind Admins eine regulaere Collection.
Der korrekte Auth-Endpoint lautet:

  curl -s -X POST http://localhost:8080/api/collections/_superusers/auth-with-password \
    -H "Content-Type: application/json" \
    -d '{"identity":"admin@example.com","password":"changeme123"}'

HINWEIS: Die alten Endpoints /api/admins/auth-with-password und
/api/collections/users/auth-with-password geben 404 zurueck — in v0.36.x normal.

## 🔄 Sync-Architektur

Die App ist fuer Offline-First / Multi-Device-Sync ausgelegt:

| Feld        | Funktion                                                    |
|-------------|-------------------------------------------------------------|
| uuid        | Geraeteuebergreifende eindeutige ID (unabh. von PB-ID)      |
| etag        | Versionshash zur Konflikterkennung beim Sync                |
| device_id   | Welches Geraet hat den Record zuletzt geaendert             |
| deleted     | Soft-Delete — kein hartes Loeschen, fuer Sync-Konsistenz   |
| updated_at  | Unix-Timestamp fuer Delta-Sync (nur geaenderte Records)     |
| remote_path | Pfad nach erfolgreichem Upload zum Server                   |
